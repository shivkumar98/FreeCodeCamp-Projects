Encapsulation

Encapsulation is a key design principle from object-oriented design

Conceptually, it means classes should wrap data and methods which operate on its data within the class itself

This is achieved by using access modifiers and making fields private, and getters/setters public

class Anime {
  private String title;
  private Genre genre;
  // public setters and getters
}

Encapsulation achieves the following:

  • Data-integrity
  • Security
Arrays

Arrays are a funamental data structure in Java

Arrays are ordered and have a fixed size

Arrays is also an interface, it can be used to store primitives and objects

int[] arr = new int[3]

Arrays have the following properties:

  • They have fast read and write speeds
  • You can get the size of an array using .lenght() method
  • Each item of the array has an index which is required to retrieve the item
Strings

Strings are an immutable data-structure in Java

String literals are managed by the String Pool which is a stored in the Heap memory

It is better for performance if we declare Strings as literals rather than using a String constructor

String str = "Hello world!"
Exceptions

Exceptions are a way of handling issues which may arrise during execution

Exceptions are either thrown programatically or by the JVM

We can handle exceptions using try/catch blocks

Polymorphism

Polymorphism derives from a Greek word which means "many forms"

Java supports Polymorphism; objects can be referenced by the class of the object OR a superclass of the object OR an interface of the class which it implements.

Polymorphism consequentially allows for method overriding in which a method's implementation is dependent on the object which calls it

class Fruits {}; class Apple extends Fruite {}; Fruits apple = new Apple(); // polymorphism
